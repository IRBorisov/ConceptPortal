Sandbox First-User Onboarding

A visual, interactive introduction that teaches users how Portal schemas and models work by guiding them through the real Sandbox editor.

Goal

Convert the first Sandbox visit into a safe guided experience. The user should leave understanding that a schema defines concepts and relationships, while a model provides concrete values and evaluation results.

System Structure

features/onboarding

Owns the welcome dialog, step controller, popover, spotlight, content, and tests.

GlobalOnboarding

Mounted near global dialogs/tooltips so it can react to routes without living inside Sandbox state.

SandboxTabs anchors

Adds stable data-onboarding-id targets to tab chrome and key editor areas.

Existing editors

The tour teaches through current RSForm/RSModel components instead of replacing them.

Key Decisions

  • Use a small custom tour layer based on existing Base UI popover primitives.
  • Keep long onboarding prose in per-locale feature files, not i18n message maps.
  • Persist completion with an onboarding version so later UX changes can re-trigger the tour.
  • Make mutation steps optional in v1 so the tutorial cannot trap users in an invalid state.
  • Do not change Sandbox bundle persistence, schema APIs, model APIs, or starter data format.

Acceptance Criteria

  • First-time users see a skippable welcome entry for Sandbox onboarding.
  • Starting the tour routes to Sandbox and highlights the current step target.
  • Next and Back can move across the Passport, List, Concept, Graph, Data, and Evaluation tabs.
  • Skip and Done persist and prevent the first-run flow from reappearing.
  • The tour remains keyboard accessible and dismissible by Escape.
  • Locale parity tests pass after adding onboarding strings.

Testing Plan

State

Unit test version handling, resume point, skip, done, and re-show behavior.

Flow

Playwright: first visit, Start to Sandbox, advance across steps, Skip persists.

Regression

Verify completed users do not see onboarding and existing Sandbox editing still works.

Scope Boundary

The onboarding system should teach through the current editor. It should not change Sandbox persistence, schema/model mutation contracts, starter bundle format, or production editor behavior beyond stable anchor attributes needed for the tour.
