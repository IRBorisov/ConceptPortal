# Доменная справка

Термины, используемые в Concept Portal.

## Основные термины

- **Предметная область** — область, для которой строится система определений.
- **Концептуальная схема (`КС`)** — система понятий, утверждений и связей между ними.
- **Концептуальная модель (`КМ`)** — КС плюс интерпретация ее понятий на конечных данных.
- **Операционная схема синтеза (`ОСС`)** — граф загрузок и синтезов КС.
- **ЯРЭ** (язык родоструктурной экспликации) — формальный язык для понятий, отношений, операций и утверждений.
- **Типизация / ступень** — структура значений выражения: элемент, целое, кортеж, множество, `Logic`.
- **Граф термов** — зависимости: конституента ссылается на другую в формальном определении.

## Классификация понятий КС

- **Неопределяемое понятие** — конституента, вводимая конвенцией (и при необходимости аксиомами), а не формальным определением. В родоструктурной экспликации: `X#`, `C#`, `S#`. Смысл задают `convention`, интерпретация и аксиомы.
- **Производное понятие** — конституента с формальным определением на ЯРЭ: `D#`, `F#`, `P#` и связанные выводимые конструкции. Значение получают из определения при заданной интерпретации неопределяемых понятий.
- **Ядро КС** — неопределяемые понятия, аксиомы и производные понятия, необходимые для их формулировки.
- **Тело КС** — остальные конституенты схемы, не входящие в ядро.
- **Конвенция** — текстовое соглашение о том, как неопределяемое понятие соотносится с сущностями предметной области. В данных схемы хранится в поле `convention` конституенты. Конвенция не заменяет формальное определение и не проходит формальную проверку ЯРЭ. Она фиксирует предметный смысл, который нельзя вывести из формулы: что именно считается элементом множества, как читать компоненты кортежа, какой порядок в паре отношения, какие константы или операции подразумеваются. В конвенции используется предметный язык и термины, которые могут быть не выражены в концептуальной схеме.
- **Интерпретация** — соотнесение понятий КС с сущностями предметной области. В концептуальной модели задаётся значениями неопределяемых конституент и вычислением производных.
- **Биективная переносимость** — независимость значения формального определения от биективной замены интерпретаций неопределяемых понятий. Для `X#` опирается на несравнимость элементов; анализатор ЯРЭ отсекает формулы, нарушающие переносимость.

## Конституенты

**Конституента** — часть КС. Поля и правила валидации см. в [CONSTITUENTA.md](CONSTITUENTA.md); здесь только предметные роли.

| Обозначение | Роль                                                                                    |
| :---------- | :-------------------------------------------------------------------------------------- |
| `N#`        | **Номиноид**: словарная сущность вне родоструктурной экспликации                        |
| `X#`        | **Базисное множество**: неопределяемые предметные объекты без порядка и арифметики      |
| `C#`        | **Константное множество**: неопределяемый конечный домен с порядком и арифметикой языка |
| `S#`        | **Родовая структура**: неопределяемое структурное понятие; формула задаёт типизацию     |
| `D#`        | **Терм**: производное понятие, вычисляемое по формуле                                   |
| `F#`        | **Терм-функция**: параметризованный терм                                                |
| `P#`        | **Предикат-функция**: параметризованное логическое выражение                            |
| `A#`        | **Аксиома**: логическое требование к модели                                             |
| `T#`        | **Высказывание**: логическое утверждение о модели                                       |

`Z` — встроенная ступень целых в ЯРЭ, **не конституента**. Для выбора между `X#`, `C#` и `Z` — [BASE-SELECTION.md](BASE-SELECTION.md).

`R#` — **радикал**, шаблонная ступень в доменах параметров `F#` / `P#`. Это тоже **не конституента**: в схеме нет записи с `alias` `R1`, `R2`, … и добавлять её не нужно ([SYNTAX.md](SYNTAX.md), раздел «Радикалы»).

Правило: `X#`, `C#`, `S#` задаются конвенцией и интерпретацией; `D#`, `F#`, `P#`, `A#`, `T#` выводятся по формуле. Поставщики должны идти раньше потребителей.

Не путай `S#` и `D#`: `S1: ℬ(X1×X1)` — интерпретируемое отношение из пар; `D1 := X1×X1` — декартово произведение.

## Зависимости

- **Поставщики** — конституенты, на которые ссылается текущая.
- **Потребители** — конституенты, которые ссылаются на текущую.
- **Влияющие** — прямые и транзитивные поставщики.
- **Зависимые** — прямые и транзитивные потребители.
- **Основание** — конституента, из которой формально порождают новые термы.
- **Порожденная конституента** — построена из основания без новой предметной договоренности.

## ОСС и синтез

- **Загрузка** добавляет внешнюю или собственную КС в ОСС.
- **Синтез** строит новую КС из операций-аргументов через таблицу отождествлений.
- **Наследованные конституенты** пришли из исходных схем; их определения обычно read-only.
- **Сквозные изменения** идут от исходной конституенты к наследникам.
- **Ромбовидный синтез** использует КС с общими предками; чаще требует ручного отождествления дублей.
- **Концептуальный блок** — группа схем внутри предметной области.

### Термины: не путать (для агентов)

В ответах пользователю и в конвенциях КС используй только термины из этого файла. Не вводи собственные ярлыки («базис», «базисный слой», «каркас схемы») — они не закреплены в Portal и размывают различия ниже.

| Не используй                                            | Используй                                                                                   | Почему                                                                      |
| :------------------------------------------------------ | :------------------------------------------------------------------------------------------ | :-------------------------------------------------------------------------- |
| «базис», «базисный слой», «первый слой» (без уточнения) | **неопределяемые понятия** — `X#`, `C#`, `S#`                                               | Общий термин для всего неопределяемого слоя                                 |
| «базис» / «базисное» про `C#` или `S#`                  | **константное множество** (`C#`), **родовая структура** (`S#`)                              | **Базисное множество** — только про `X#`                                    |
| «базисные множества» как синоним неопределяемых понятий | **базисные множества** — только несколько `X#`; для всего слоя — **неопределяемые понятия** | `C#` и `S#` базисными множествами не называются                             |
| «ядро» = неопределяемые понятия»                        | **ядро КС** — неопределяемые понятия плюс аксиомы и производные, нужные для аксиом          | Ядро шире, чем только `X#` / `C#` / `S#`                                    |
| теорема про `T#`                                        | **высказывание** — производное логическое выражение `T#`                                    | тип теорема на Portal отсутствует и вводит в заблуждение                    |
| «пропущенная конституента `R1`» в списке схемы          | **радикал** `R#` — параметр шаблона в домене `F#` / `P#`                                    | `R#` не объявляется; при ревью КС отсутствие `R1` среди конституент — норма |
| «добавить `R1` как базисное / структурное понятие»      | оставить `R#` только в `[…]` у параметризованных `F#` / `P#`                                | иначе смешиваются шаблон и предметная КС                                    |

## Статусы выражений

- `Unknown` — не проверено.
- `OK` — корректно.
- `Error` — ошибка в определении.
- `Property` — неразмерное: доступна только проверка принадлежности.
- `Incalculable` — невычислимо напрямую.

## Статусы вычисления в модели

- `NO_EVAL` — нечего вычислять.
- `NOT_PROCESSED` — вычисление не запускалось.
- `INVALID_DATA` — неверная интерпретация.
- `EVAL_FAIL` — ошибка вычисления.
- `AXIOM_FALSE` — аксиома ложна.
- `EMPTY` — результат пуст.
- `HAS_DATA` — результат вычислен и непустой.
