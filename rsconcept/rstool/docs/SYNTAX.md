# Синтаксис RSLang

Единая справка для агента: читай при построении или исправлении `definitionFormal`. Типизация — `TYPIFICATION.md`; ошибки — `DIAGNOSTICS.md`.

## Имена и литералы

- Глобальные алиасы: `X1`, `C2`, `S3`, `D11`, `F7`, `P4`, `A6`, `T9`; буква совпадает с ролью конституенты.
- Радикалы: `R1`, `R2` — шаблонные ступени в параметрах. Не путать с префиксом рекурсии `R{…}`.
- Локальные переменные: `x`, `ξ`, `μ2`, `y1`.
- Целые: `0`, `42`; отрицательного литерала нет, пиши `0 - 5`.
- `Z` — множество целых, `∅` — пустое множество с типизацией `ℬ(R0)`.
- Скобки и разделители: `()`, `[]`, `{}`, `,`, `;`, `|`.
- Комментариев в формальных выражениях нет.

## Теоретико-множественные выражения (STE)

- `D1 ∪ D2`, `D1 ∩ D2`, `D1 \ D2`, `D1 ∆ D2` — операции над множествами.
- `X1 × X2` — декартово произведение.
- `ℬ(X1)` — булеан, множество всех подмножеств.
- `(a, b, c)` — кортеж, арность минимум 2.
- `{a, b, c}` — перечисление.
- `bool(a)` — синглетон, `debool(S)` — извлечение единственного элемента.
- `red(S)` — суммарное множество; `S` должно иметь ступень `ℬ(ℬ(H))`.
- `pr1(tuple)` — малая проекция кортежа.
- `Pr1(S)` — большая проекция множества кортежей.
- `Fi1[D](S)` — фильтр по принадлежности выбранной проекции в `D`.

Мультииндексы разрешены: `pr1,3(...)`, `Pr2,4(S1)`, `Fi1,2[D1](S1)`.

## Генераторы: `D{}`, `I{}`, `R{}`

Три конструкции строят STE. Буквы `D`, `I`, `R` перед `{` — **часть синтаксиса**, не алиасы конституент.

| Конструкция | Результат                 | Когда использовать                                                            |
| :---------- | :------------------------ | :---------------------------------------------------------------------------- |
| `D{…}`      | множество `ℬ(H)`          | отбор по условию из уже заданного множества (родо-видовое определение)        |
| `I{…}`      | множество `ℬ(H)`          | то же множество, но через последовательность переборов, присвоений и проверок |
| `R{…}`      | одно значение ступени `H` | итеративный пересчёт до выхода или стабилизации                               |

### Декларативная `D{переменная ∈ домен | условие}`

Схема ограниченного выделения: перебирает `домен`, включает в результат каждый элемент (или кортеж), для которого `условие` истинно.

```text
D{ξ∈D1 | LE(ξ)}
D{(ξ1, ξ2)∈S1 | LE(ξ1, ξ2)}
{ξ∈D1 | LE(ξ)}          — краткая форма без префикса D, только одна переменная
```

- Слева от `|` — объявление переменной и домен (`∈`, не `:∈`).
- Справа — логическое выражение (`LE`).
- Типизация результата: `ℬ(H)`, где `H` — ступень элементов домена.
- Кортеж в объявлении требует префикса `D`: `D{(a, b)∈X1 | …}`.
- Пример: `D{ξ∈{1, 2, 3, 4, 5, 6} | ∃σ∈{10, 11, 12} σ = 2 * ξ}` → `{5, 6}`.

### Императивная `I{элемент | блоки через ;}`

Строит множество перебором и присвоениями. Эквивалентна декларативной, но удобна, когда значения зависят от предыдущих шагов.

```text
I{(ξ1, ξ2) | ξ1:∈D1; ξ2:=F1[ξ1]; LE(ξ1, ξ2)}
I{β | δ1:=Fi1,2[α, α](σ); δ2:=δ1\Pr1,1(δ1); β:=Pr1(δ2)\Pr2(δ2)}
```

Слева от `|` — STE, задающее элемент результата (часто кортеж локальных переменных).

Справа — блоки через `;`. Три вида блока:

| Блок       | Синтаксис | Смысл                                                            |
| :--------- | :-------- | :--------------------------------------------------------------- |
| Перебор    | `ξ:∈STE`  | `ξ` (или кортеж) перебирает элементы множества                   |
| Присвоение | `ξ:=STE`  | `ξ` получает вычисленное значение                                |
| Проверка   | `LE`      | при ложности дальнейшие блоки пропускаются, перебор продолжается |

- Префикс `I` обязателен; в переборе — `:∈`, не `∈`.
- Типизация результата: `ℬ(H)`, где `H` — ступень выражения слева от `|`.
- Пример: `I{(ξ1, ξ2) | ξ1:∈{1, 2, 3}; ξ2:=ξ1+1; ∃σ∈{4, 5, 6} σ=2*ξ2}` → `{(1, 2), (2, 3)}`.

### Рекурсивная `R{переменная := база | условие продолжения | формула следующего шага}`

Итеративно пересчитывает переменную рекурсии; возвращает **одно значение**, не множество.

```text
R{ξ := STE1 | STE2(ξ)}                    — короткая форма
R{ξ := STE1 | LE(ξ) | STE2(ξ)}            — полная форма с условием продолжения
R{(ξ, σ) := (STE1, STE2) | LE(ξ, σ) | (STE3, STE4)}
```

| Часть               | Позиция             | Смысл                                                             |
| :------------------ | :------------------ | :---------------------------------------------------------------- |
| База                | `переменная := STE` | начальное значение                                                |
| Условие продолжения | `LE` (опционально)  | пересчитывается на каждом шаге; при ложности цикл останавливается |
| Пересчёт            | последний `STE`     | новое значение переменной на следующем шаге                       |

Порядок шага: присвоить текущее значение → (если есть) проверить условие → вычислить пересчёт.

Завершение: условие продолжения ложно, либо значение на шаге `k` равно значению на шаге `k+1` (в том числе в короткой форме без `LE`).

- Префикс `R` обязателен; в присвоении — `:=`.
- Типизация результата совпадает со ступенью переменной рекурсии; пересчёт должен иметь ту же ступень.
- Пример: `R{(ξ, σ) := (1, 0) | σ<5 | (2*ξ, σ+1)}` → `32`.

## Логические выражения (LE)

- Предикаты множеств: `ξ ∈ S`, `ξ ∉ S`, `S1 = S2`, `S1 ≠ S2`, `S1 ⊆ S2`, `S1 ⊂ S2`, `S1 ⊄ S2`.
- Арифметические предикаты: `=`, `≠`, `<`, `≤`, `>`, `≥` над `Z`.
- Связки: `¬A`, `A & B`, `A ∨ B`, `A ⇒ B`, `A ⇔ B`.

`TRUE` и `FALSE` в экспликации КС не используют.

## Кванторы

- `∀ξ∈D1 (LE(ξ))` — всеобщность.
- `∃ξ∈D1 (LE(ξ))` — существование.
- `∀(ξ1, ξ2)∈S1 (LE(ξ1, ξ2))` — объявление кортежа.
- `∀ξ1, ξ2∈D1 (LE(ξ1, ξ2))` — сокращение вложенных кванторов.

Скобки вокруг `LE` можно опустить для атомарного логического выражения.

## Арифметика

`a + b`, `a - b`, `a * b` дают `Z`. `card(S)` возвращает мощность конечного множества.

## Параметризованные выражения

```text
F1 ::= [α1∈STE1, α2∈STE2(α1)] STE(α1, α2)
P1 ::= [α1∈STE1, α2∈STE2(α1)] LE(α1, α2)
```

- В `[]` идут параметры через `∈`.
- Параметр доступен в следующих доменах и теле.
- Вызов позиционный: `F1[ξ1, S1]`, `P1[ξ1\ξ2, ξ3]`.
- `F#` возвращает STE, `P#` — LE.

## Радикалы

```text
F2 ::= [α1∈R1×R2, α2∈R1] α2 = pr1(α1)
```

- `R#` выводится по типизациям аргументов в месте вызова.
- Все вхождения одного `R#` должны совпасть.
- Радикалы допустимы только в доменах параметров.

## Примеры

- `¬α ∈ S1`
- `D1 ⊆ D2 ⇔ ∀x∈D1 x∈D2`
- `∀x∈D1 ∃y∈D2 (x, y)∈S1 & (x, x)∈S1`
- `(4 + 5) * 3`, `card(X1) > 5`
- `pr2((5, 4, 3, 2, 1)) = 4`
- `red({{1, 2, 3}, {3, 4}}) = {1, 2, 3, 4}`
- `D{ξ∈σ | card(ξ) > 2}`
- `I{(ξ1, ξ2) | ξ1:∈Pr1(σ); ξ2:∈Pr2(Fi1[{ξ1}](σ))}`
- `R{ξ:=Pr2(Fi1[α](σ)) | ξ ∪ Pr2(Fi1[ξ](σ))}`

## Корректность

- Анализ идет в глобальном контексте: все алиасы должны быть известны и типизированы.
- Формальные определения должны быть биективно переносимыми (гарантируется семантическим анализатором).
- Логическая непротиворечивость проверяется примером интерпретации в модели.

## Приоритеты операторов

От высшего к низшему:

1. Вызовы функций, проекции, `ℬ`, `bool`, `debool`, `red`, `card`.
2. `×`.
3. `*`.
4. `+`, `-`.
5. `∪`, `∩`, `\`, `∆`.
6. Предикаты: `∈`, `=`, `<`, …
7. `¬`.
8. `&`.
9. `∨`.
10. `⇒`.
11. `⇔`.
